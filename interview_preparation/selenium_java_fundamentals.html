1. Selenium Basics & Introduction

Core concepts every candidate must know
1 What is Selenium? What are its components?
Selenium is an open-source suite of tools for automating web browsers. It supports multiple languages (Java, Python, C#, etc.) and browsers (Chrome, Firefox, Safari, Edge).

Four components:
  • Selenium WebDriver — The core API that directly communicates with the browser through its native driver (ChromeDriver, GeckoDriver). This is what most test frameworks use.
  • Selenium IDE — A browser extension for recording and playback of tests. Good for beginners, not suitable for production use.
  • Selenium Grid — Allows running tests in parallel across multiple machines and browsers simultaneously.
  • Selenium RC (deprecated) — The old server-based approach. Replaced by WebDriver.
2 What is WebDriver? How does it work?
WebDriver is an interface in Selenium that provides methods to control a browser programmatically. It communicates with the browser using the W3C WebDriver Protocol (HTTP-based JSON Wire Protocol).

How it works:
  • Your Java code calls a WebDriver method (e.g., driver.findElement())
  • The Selenium client library converts this to an HTTP request
  • The browser driver (ChromeDriver) receives the request and executes it in the browser
  • The browser sends the result back via HTTP response
Example: ChromeDriver is a standalone server that implements the W3C protocol and acts as a bridge between your test code and Chrome.
3 What are the limitations of Selenium?
  • Cannot automate desktop/native applications — only web browsers
  • No built-in reporting — need external tools (Allure, ExtentReports)
  • Cannot handle CAPTCHA by design (security feature)
  • Requires browser driver updates when the browser updates
  • Limited support for image-based verification
  • Flakiness due to dynamic web content and timing issues
  • Cannot automate mobile apps directly — needs Appium for that
4 What is the difference between Selenium 3 and Selenium 4?

5 What browsers does Selenium support?
Selenium supports: Chrome (ChromeDriver), Firefox (GeckoDriver), Edge (EdgeDriver), Safari (SafariDriver — built into Safari on Mac), Internet Explorer (IEDriverServer — legacy).

Each browser needs its corresponding driver executable that matches the browser version.

7 Write a simple first Selenium test in Java.
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.chrome.ChromeDriver;
import io.github.bonigarcia.wdm.WebDriverManager;
import org.testng.Assert;
import org.testng.annotations.*;

public class FirstTest {
    WebDriver driver;

    @BeforeMethod
    public void setUp() {
        WebDriverManager.chromedriver().setup(); // auto-downloads ChromeDriver
        driver = new ChromeDriver();
        driver.manage().window().maximize();
    }

    @Test
    public void verifyPageTitle() {
        driver.get("https://www.google.com");
        String title = driver.getTitle();
        Assert.assertEquals(title, "Google");
    }

    @AfterMethod
    public void tearDown() {
        if (driver != null) {
            driver.quit();
        }
    }
}
Explanation: @BeforeMethod runs before each test to create the driver. @AfterMethod runs after each test to close the browser. driver.quit() closes all browser windows and ends the driver session.
8 What is the difference between driver.close() and driver.quit()?
  • driver.close() — Closes only the currently focused browser window/tab. The WebDriver session remains active. Use this when you open multiple windows and want to close just one.
  • driver.quit() — Closes all browser windows and completely terminates the WebDriver session. Always use this in @AfterMethod/teardown to avoid browser processes lingering in memory.
Common mistake: Using close() instead of quit() in teardown leaves ChromeDriver processes running in the background, consuming memory.
9 What is WebDriverManager and why is it used?
WebDriverManager (by Boni Garcia) is a library that automatically downloads and manages browser driver binaries (ChromeDriver, GeckoDriver, etc.) matching the installed browser version.

Without WebDriverManager: You must manually download ChromeDriver, set the system property, and update it every time Chrome updates.
With WebDriverManager: 
WebDriverManager.chromedriver().setup(); // one line — done
WebDriver driver = new ChromeDriver();
This is now standard practice in all modern Selenium Java projects.

3. Locators — Finding Elements

The most tested topic in Selenium interviews
10 What are the different types of locators in Selenium?
LocatorMethodBest used when
IDBy.id("username")Element has a unique, stable ID — fastest and most reliable
NameBy.name("email")Form elements with a name attribute
Class NameBy.className("btn-primary")Single class; fails if multiple classes
Tag NameBy.tagName("input")Finding all elements of a type
Link TextBy.linkText("Click Here")Anchor tags with exact visible text
Partial Link TextBy.partialLinkText("Click")Anchor tags with partial text
CSS SelectorBy.cssSelector("input#email")Fast, flexible, works for most cases
XPathBy.xpath("//input[@id='email']")Complex conditions, traversal, text matching
Priority order: ID > Name > CSS Selector > XPath. Avoid XPath unless CSS can't do it — XPath is slower and more brittle.
11 Explain CSS Selector with examples.
CSS Selectors use CSS syntax to locate elements. They are faster than XPath and widely preferred. 
// By tag and ID
By.cssSelector("input#username")         // <input id="username">

// By tag and class
By.cssSelector("button.btn-primary")     // <button class="btn-primary">

// By attribute
By.cssSelector("input[type='submit']")   // <input type="submit">

// Attribute starts with
By.cssSelector("input[id^='user']")      // id starts with "user"

// Attribute ends with
By.cssSelector("input[id$='name']")      // id ends with "name"

// Attribute contains
By.cssSelector("input[id*='ser']")       // id contains "ser"

// Parent > direct child
By.cssSelector("div.form > input")

// nth child
By.cssSelector("table tr:nth-child(2)")  // 2nd row
12 Explain XPath with examples — absolute vs relative.
Absolute XPath — starts from root, follows the full path. Brittle — breaks if any element in the path changes. 
// Absolute — avoid this
/html/body/div[1]/form/input[2]
Relative XPath — starts with //, searches anywhere in the DOM. Preferred. 
// By attribute
//input[@id='username']
//button[@type='submit']

// By text content
//button[text()='Login']
//a[contains(text(),'Sign')]

// By multiple attributes
//input[@type='text' and @name='email']

// Parent to child
//div[@class='form']//input

// Child to parent (XPath only — CSS can't do this)
//input[@id='username']/..        // goes up to parent
//input[@id='username']/parent::div

// Following sibling
//label[text()='Email']/following-sibling::input
13 What is the difference between findElement() and findElements()?
  • findElement(By locator) — Returns a single WebElement. Throws NoSuchElementException if the element is not found.
  • findElements(By locator) — Returns a List<WebElement>. Returns an empty list (not an exception) if nothing is found. Use this to check if an element exists, or to get all matching elements.
// Check if element exists safely
boolean exists = !driver.findElements(By.id("error-msg")).isEmpty();

// Get all rows in a table
List<WebElement> rows = driver.findElements(By.cssSelector("table tr"));
System.out.println("Row count: " + rows.size());
14 How do you find a dynamic element whose ID changes every time?
Dynamic IDs cannot be used directly. Use these alternatives:
  • Use a stable attribute: data-testid, name, class
  • CSS partial match: By.cssSelector("[id^='prefix_']") — if ID has a stable prefix
  • Text-based XPath: //button[contains(text(),'Submit')]
  • Parent-child relationship: navigate via a stable parent element
  • Ask the dev team to add a data-testid attribute — the right long-term fix

16 What is the difference between get() and navigate().to()?
Both open a URL and wait for the page to load. The difference:
  • driver.get() — Opens a URL. Does not maintain browser history in the session.
  • driver.navigate().to() — Opens a URL AND maintains browser history, allowing navigate().back() and navigate().forward() to work.
In practice, use navigate().to() when you need back/forward navigation in your test. Otherwise both are equivalent.
17 How do you take a screenshot in Selenium?
import org.apache.commons.io.FileUtils;
import org.openqa.selenium.OutputType;
import org.openqa.selenium.TakesScreenshot;

// Cast driver to TakesScreenshot
TakesScreenshot ts = (TakesScreenshot) driver;

// Get screenshot as File
File srcFile = ts.getScreenshotAs(OutputType.FILE);

// Save to disk
File destFile = new File("screenshots/test_" + System.currentTimeMillis() + ".png");
FileUtils.copyFile(srcFile, destFile);

// As byte[] — for embedding in Allure/TestNG reports
byte[] bytes = ts.getScreenshotAs(OutputType.BYTES);

// Element-level screenshot (Selenium 4 only)
WebElement element = driver.findElement(By.id("chart"));
File elShot = element.getScreenshotAs(OutputType.FILE);
Best practice: Call screenshot in @AfterMethod only when the test has failed — check ITestResult.getStatus() == ITestResult.FAILURE.
18 How do you use JavascriptExecutor? When do you need it?
JavascriptExecutor executes JavaScript directly in the browser. Use it when Selenium's native methods don't work (hidden elements, scroll, readonly fields). 
// Cast driver
JavascriptExecutor js = (JavascriptExecutor) driver;

// Scroll to bottom of page
js.executeScript("window.scrollTo(0, document.body.scrollHeight);");

// Scroll element into view
WebElement el = driver.findElement(By.id("footer"));
js.executeScript("arguments[0].scrollIntoView(true);", el);

// Click via JS (when normal click fails)
js.executeScript("arguments[0].click();", el);

// Set value on readonly input
js.executeScript("arguments[0].value='2026-01-01';", dateInput);

// Get hidden element text
String text = (String) js.executeScript("return arguments[0].innerText;", el);

// Check page load complete
String state = (String) js.executeScript("return document.readyState;");
// returns "complete" when page is fully loaded

5. Waits — Implicit, Explicit, Fluent

The most important concept for stable tests — heavily tested in interviews
20 Why do we need waits in Selenium?
Web applications are dynamic — elements load asynchronously (via AJAX, API calls, animations). Without waits, Selenium may try to interact with an element before it exists in the DOM, causing NoSuchElementException or ElementNotInteractableException.

Waits tell Selenium to pause and check until a condition is met, making tests stable.
Never use Thread.sleep() — it always waits the full time even if the element loads early. It's a hardcoded pause, not a smart wait.
21 Explain Implicit Wait.
Implicit wait sets a global timeout that applies to every findElement() call. If the element isn't immediately found, Selenium polls the DOM repeatedly until the timeout expires. 
// Set globally once — affects all findElement calls
driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

// Now this won't fail immediately if element takes a few seconds to appear
WebElement el = driver.findElement(By.id("username"));
Limitation: It applies to ALL findElement calls and cannot be overridden for specific elements. Cannot wait for custom conditions (like element to be clickable).
Best practice: Set implicit wait to a low value (3-5s) or don't use it at all — use Explicit Wait instead for precise control.
22 Explain Explicit Wait with ExpectedConditions.
Explicit wait waits for a specific condition on a specific element before proceeding. More precise and powerful than implicit wait. 
import org.openqa.selenium.support.ui.WebDriverWait;
import org.openqa.selenium.support.ui.ExpectedConditions;

WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(15));

// Wait until element is visible in DOM
WebElement el = wait.until(
    ExpectedConditions.visibilityOfElementLocated(By.id("result"))
);

// Wait until element is clickable (visible + enabled)
wait.until(ExpectedConditions.elementToBeClickable(By.id("submit"))).click();

// Wait until text appears in element
wait.until(ExpectedConditions.textToBePresentInElementLocated(
    By.id("status"), "Success"
));

// Wait until element disappears (loading spinner)
wait.until(ExpectedConditions.invisibilityOfElementLocated(By.id("spinner")));

// Wait for URL to change
wait.until(ExpectedConditions.urlContains("dashboard"));

// Wait for title
wait.until(ExpectedConditions.titleContains("Home"));
23 Explain Fluent Wait and how it differs from Explicit Wait.
Fluent Wait is an advanced form of Explicit Wait with more configuration options: 
import org.openqa.selenium.support.ui.FluentWait;

Wait<WebDriver> fluentWait = new FluentWait<>(driver)
    .withTimeout(Duration.ofSeconds(30))         // max wait time
    .pollingEvery(Duration.ofMillis(500))        // check every 500ms
    .ignoring(NoSuchElementException.class)     // ignore this exception while polling
    .ignoring(StaleElementReferenceException.class)
    .withMessage("Button not found after 30 seconds"); // custom error message

WebElement btn = fluentWait.until(
    ExpectedConditions.elementToBeClickable(By.id("submit"))
);
FeatureExplicit WaitFluent Wait
Polling intervalFixed (500ms default)Configurable
Exception ignoringNot configurableConfigurable
Custom messageNot supportedSupported
Use caseMost scenariosSlow-loading or unstable elements
24 Why should you not mix Implicit and Explicit waits?
Mixing both causes unpredictable behavior and double-wait times. When both are active:
  • Implicit wait is global and fires on every findElement()
  • Explicit wait has its own polling loop
  • If the element doesn't exist, Selenium may wait for the full implicit wait duration on each polling cycle of the explicit wait — causing very long wait times
Rule: Pick one strategy. Prefer Explicit Wait for professional frameworks. If you use implicit wait, keep it very short (2-3s) and don't mix with explicit waits on the same elements.

6. Handling Web Elements

Dropdowns, Alerts, Frames, Multiple Windows — all frequently asked
25 How do you handle a dropdown (<select> element) in Selenium?
Use the Select class for HTML <select> dropdowns. 
import org.openqa.selenium.support.ui.Select;

WebElement dropdownEl = driver.findElement(By.id("country"));
Select select = new Select(dropdownEl);

// Three ways to select an option
select.selectByVisibleText("United States");  // by the displayed text
select.selectByValue("us");                    // by the 'value' attribute
select.selectByIndex(2);                         // by position (0-based)

// Get all options
List<WebElement> options = select.getOptions();
options.forEach(o -> System.out.println(o.getText()));

// Get selected option
String selected = select.getFirstSelectedOption().getText();

// Multi-select dropdown
select.selectByVisibleText("Option A");
select.selectByVisibleText("Option B");
select.deselectAll(); // only works if isMultiple()
Note: The Select class only works with native HTML <select> tags. For custom dropdowns (built with div/ul), click the element and then click the option as a normal WebElement.
26 How do you handle alerts in Selenium?
JavaScript alerts/confirms/prompts are handled using driver.switchTo().alert(). 
// Wait for alert to appear
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(5));
Alert alert = wait.until(ExpectedConditions.alertIsPresent());

// Get alert message
String message = alert.getText();
System.out.println("Alert says: " + message);

// Simple alert — just OK button
alert.accept();   // click OK

// Confirm dialog — OK and Cancel
alert.accept();   // click OK
alert.dismiss();  // click Cancel

// Prompt — enter text before accepting
alert.sendKeys("My input text");
alert.accept();
Common mistake: Trying to use findElement() inside an alert. Alerts are not DOM elements — always use the Alert API.
27 How do you switch between iframes?
An iframe is an embedded HTML page within a page. WebDriver cannot directly interact with elements inside an iframe — you must switch context first. 
// Switch to iframe by index (0-based)
driver.switchTo().frame(0);

// Switch by name or ID attribute
driver.switchTo().frame("iframeName");

// Switch by WebElement (most reliable)
WebElement iframeEl = driver.findElement(By.cssSelector("iframe.payment-frame"));
driver.switchTo().frame(iframeEl);

// Now interact with elements inside the iframe
driver.findElement(By.id("card-number")).sendKeys("4111111111111111");

// Switch BACK to the main page
driver.switchTo().defaultContent();

// Switch to parent frame (when iframes are nested)
driver.switchTo().parentFrame();
28 How do you handle multiple browser windows or tabs?
// Store handle of current (main) window
String mainWindow = driver.getWindowHandle();

// Click something that opens a new tab/window
driver.findElement(By.linkText("Open New Tab")).click();

// Get all open window handles
Set<String> allWindows = driver.getWindowHandles();

// Switch to the new window
for (String handle : allWindows) {
    if (!handle.equals(mainWindow)) {
        driver.switchTo().window(handle);
        break;
    }
}

// Interact with new window
System.out.println("New window title: " + driver.getTitle());

// Close new window and switch back
driver.close();
driver.switchTo().window(mainWindow);
29 How do you handle file uploads?
// For <input type="file"> — use sendKeys with file path
WebElement fileInput = driver.findElement(By.cssSelector("input[type='file']"));
fileInput.sendKeys("C:\\Users\\user\\Documents\\test.pdf");

30 How do you handle checkboxes and radio buttons?
// Checkbox — check if selected before clicking
WebElement checkbox = driver.findElement(By.id("agree"));
if (!checkbox.isSelected()) {
    checkbox.click();  // select it
}

// Uncheck it
if (checkbox.isSelected()) {
    checkbox.click();
}

// Radio button — just click (selecting one deselects others)
driver.findElement(By.cssSelector("input[value='male']")).click();

// Verify selection
boolean isChecked = driver.findElement(By.id("agree")).isSelected();
Assert.assertTrue(isChecked, "Checkbox should be checked");

7. Page Object Model (POM)

Design pattern that separates page elements from test logic
31 What is the Page Object Model? Why do we use it?
POM is a design pattern where each web page (or major component) has a corresponding Java class. That class contains:
  • All locators for elements on that page
  • Methods that represent actions on that page (login, search, submit)
Why use POM?
  • Maintainability: If a locator changes, update it in ONE place (the page class), not in every test
  • Reusability: Methods can be reused across multiple tests
  • Readability: Tests read like plain English — loginPage.login("user", "pass")
  • Separation of concerns: Test logic is separate from page interaction logic
32 Show a complete POM example — LoginPage and LoginTest.
// LoginPage.java — Page Object
public class LoginPage {
    private WebDriver driver;
    private WebDriverWait wait;

    // Locators
    private By usernameField = By.id("username");
    private By passwordField = By.id("password");
    private By loginButton   = By.cssSelector("button[type='submit']");
    private By errorMessage  = By.className("error-msg");

    public LoginPage(WebDriver driver) {
        this.driver = driver;
        this.wait = new WebDriverWait(driver, Duration.ofSeconds(10));
    }

    public void enterUsername(String username) {
        wait.until(ExpectedConditions.visibilityOfElementLocated(usernameField))
            .sendKeys(username);
    }

    public void enterPassword(String password) {
        driver.findElement(passwordField).sendKeys(password);
    }

    public DashboardPage clickLogin() {
        driver.findElement(loginButton).click();
        return new DashboardPage(driver); // returns next page object
    }

    public String getErrorMessage() {
        return wait.until(ExpectedConditions.visibilityOfElementLocated(errorMessage))
                   .getText();
    }

    // Convenience method
    public DashboardPage login(String user, String pass) {
        enterUsername(user);
        enterPassword(pass);
        return clickLogin();
    }
}
// LoginTest.java — Test class (clean, no locators here)
public class LoginTest {
    WebDriver driver;
    LoginPage loginPage;

    @BeforeMethod
    public void setUp() {
        WebDriverManager.chromedriver().setup();
        driver = new ChromeDriver();
        driver.get("https://app.example.com/login");
        loginPage = new LoginPage(driver);
    }

    @Test
    public void validLogin() {
        DashboardPage dashboard = loginPage.login("admin", "Admin@123");
        Assert.assertTrue(dashboard.isWelcomeMessageVisible());
    }

    @Test
    public void invalidLogin() {
        loginPage.enterUsername("wrong@test.com");
        loginPage.enterPassword("wrongpass");
        loginPage.clickLogin();
        Assert.assertEquals(loginPage.getErrorMessage(), "Invalid credentials");
    }

    @AfterMethod
    public void tearDown() { driver.quit(); }
}
33 What is PageFactory and @FindBy? How does it differ from By locators?
PageFactory is a Selenium utility that initializes page objects using @FindBy annotations. 
import org.openqa.selenium.support.FindBy;
import org.openqa.selenium.support.PageFactory;

public class LoginPage {
    private WebDriver driver;

    // @FindBy annotations replace By.* locators
    @FindBy(id = "username")
    private WebElement usernameField;

    @FindBy(css = "button[type='submit']")
    private WebElement loginButton;

    @FindBy(xpath = "//div[@class='error']")
    private WebElement errorMessage;

    // FindAll — gets all matching elements
    @FindAll({
        @FindBy(className = "menu-item"),
        @FindBy(css = "nav a")
    })
    private List<WebElement> menuItems;

    public LoginPage(WebDriver driver) {
        this.driver = driver;
        PageFactory.initElements(driver, this); // REQUIRED — initializes all @FindBy fields
    }

    public void login(String user, String pass) {
        usernameField.sendKeys(user);
        loginButton.click();
    }
}
Key difference: @FindBy uses lazy initialization — elements are looked up fresh each time they are used, which helps with StaleElementReferenceException. Regular By locators also look up elements on each call but are more explicit.

8. TestNG — Test Framework

Annotations, assertions, grouping, and data providers
34 What are the key TestNG annotations and their execution order?
public class TestNGAnnotationOrder {

    @BeforeSuite   // runs once before ALL tests in the suite
    public void beforeSuite() {}

    @BeforeTest    // runs before each <test> tag in testng.xml
    public void beforeTest() {}

    @BeforeClass   // runs once before the first method in this class
    public void beforeClass() {}

    @BeforeMethod  // runs before EACH @Test method — most commonly used
    public void beforeMethod() {}

    @Test
    public void myTest() {}

    @AfterMethod   // runs after EACH @Test method
    public void afterMethod() {}

    @AfterClass    // runs once after all methods in this class
    public void afterClass() {}

    @AfterTest     // runs after each <test> tag in testng.xml
    public void afterTest() {}

    @AfterSuite    // runs once after ALL tests in the suite
    public void afterSuite() {}
}
Order: BeforeSuite → BeforeTest → BeforeClass → BeforeMethod → Test → AfterMethod → AfterClass → AfterTest → AfterSuite
35 What are TestNG assertions? Hard vs Soft assertions?
import org.testng.Assert;
import org.testng.asserts.SoftAssert;

// Hard Assert — stops test immediately on failure
Assert.assertEquals(actual, expected, "Values don't match");
Assert.assertTrue(condition, "Condition is false");
Assert.assertFalse(condition);
Assert.assertNotNull(object, "Object is null");
Assert.assertNull(object);

// Soft Assert — collects ALL failures, reports at the end
SoftAssert soft = new SoftAssert();
soft.assertEquals(page.getName(), "John", "Name wrong");
soft.assertEquals(page.getEmail(), "john@test.com", "Email wrong");
soft.assertTrue(page.isLogoutVisible(), "Logout not visible");
soft.assertAll(); // MUST call this — otherwise failures are ignored!
When to use each:
  • Hard Assert — When a failure in step N means subsequent steps are invalid (e.g., login failed → can't test anything else)
  • Soft Assert — When validating multiple independent fields on a form or page and you want all failures reported at once

Know each exception, its cause, and how to fix it
39 Explain the most common Selenium exceptions and how to handle them.
ExceptionCauseFix
NoSuchElementException Element not found in DOM with the given locator Verify locator in DevTools → use Explicit Wait → check if element is inside an iframe
StaleElementReferenceException Element was found, then the DOM refreshed (AJAX, page navigation) making the reference invalid Re-find the element; use ExpectedConditions.refreshed(); use Page Object (it re-finds on each access)
ElementNotInteractableException Element exists but is hidden, disabled, or covered by another element Wait for element to be visible/enabled; scroll into view; use JS click as last resort
ElementClickInterceptedException Click intercepted by another element (cookie banner, modal, tooltip) Dismiss the overlay first; wait for it to disappear; use JS click
TimeoutException Wait condition not met within the timeout period Increase timeout; verify the condition is correct; check if element is in iframe or new window
InvalidSelectorException XPath or CSS syntax is invalid Validate the selector in browser DevTools console first
NoAlertPresentException Trying to switch to an alert that hasn't appeared yet Use ExpectedConditions.alertIsPresent() before switching

40 What is StaleElementReferenceException and how do you fix it?
It occurs when Selenium holds a reference to an element, but the DOM changes (AJAX refresh, page navigation, Angular re-render), making that reference point to an element that no longer exists. 
// Method 1: Re-find the element
try {
    driver.findElement(By.id("btn")).click();
} catch (StaleElementReferenceException e) {
    driver.findElement(By.id("btn")).click(); // re-find and retry
}

// Method 2: Use ExpectedConditions.refreshed()
WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));
wait.until(ExpectedConditions.refreshed(
    ExpectedConditions.elementToBeClickable(By.id("btn"))
)).click();

// Method 3: Retry loop
int attempts = 0;
while (attempts < 3) {
    try {
        driver.findElement(By.id("btn")).click();
        break;
    } catch (StaleElementReferenceException e) {
        attempts++;
    }
}
41 How do you verify if an element is present on the page without throwing an exception?
// Use findElements — returns empty list, not exception
public boolean isElementPresent(By locator) {
    return !driver.findElements(locator).isEmpty();
}

// With try-catch
public boolean isElementVisible(By locator) {
    try {
        return driver.findElement(locator).isDisplayed();
    } catch (NoSuchElementException e) {
        return false;
    }
}

// Usage in test
if (isElementPresent(By.id("cookie-banner"))) {
    driver.findElement(By.id("accept-cookies")).click();
}

How to answer practical "how would you handle..." questions
42 How would you automate a login flow that uses OTP (one-time password)?
Real OTPs cannot be automated directly. Standard approaches:
  • Best approach: Ask the dev team to expose a test API endpoint that returns the OTP for test accounts. Call it in your test and enter the OTP.
  • Test environment bypass: Use a fixed OTP (e.g., 000000) for test accounts in non-production environments — configured on the backend.
  • TOTP (time-based OTP): Use a library like GoogleAuthenticator in Java to generate the same TOTP using the shared secret key.
  • Email OTP: Use a test email API (Mailinator, MailSlurp) to read the OTP email programmatically.
  • SMS OTP: Use a virtual SIM API service (Twilio test mode) to retrieve the SMS.

47 What is headless browser testing? When do you use it?
Headless mode runs the browser without a visible GUI. The browser executes normally but without rendering a window on screen. 
// Chrome headless
ChromeOptions options = new ChromeOptions();
options.addArguments("--headless=new");   // Selenium 4 / Chrome 112+
options.addArguments("--no-sandbox");
options.addArguments("--disable-dev-shm-usage");
options.addArguments("--window-size=1920,1080");
WebDriver driver = new ChromeDriver(options);

// Firefox headless
FirefoxOptions ffOptions = new FirefoxOptions();
ffOptions.addArguments("-headless");
WebDriver driver = new FirefoxDriver(ffOptions);
When to use headless:
  • CI/CD pipelines (Jenkins, GitHub Actions) — no display server available
  • Faster test execution (20-30% faster than headed)
  • Server environments (Linux without GUI)
When NOT to use: During development/debugging — you need to see the browser to diagnose failures.
48 How do you run Selenium tests in parallel?
<!-- testng.xml — parallel execution -->
<suite name="ParallelSuite" parallel="methods" thread-count="4">
  <test name="AllTests">
    <classes>
      <class name="tests.LoginTest"/>
      <class name="tests.SearchTest"/>
    </classes>
  </test>
</suite>
// CRITICAL: Use ThreadLocal to give each thread its own driver instance
public class DriverManager {
    private static ThreadLocal<WebDriver> tlDriver = new ThreadLocal<>();

    public static void setDriver(WebDriver driver) { tlDriver.set(driver); }
    public static WebDriver getDriver() { return tlDriver.get(); }
    public static void quit() { tlDriver.get().quit(); tlDriver.remove(); }
}

// BaseTest — creates driver in @BeforeMethod (called per-thread)
@BeforeMethod
public void setUp() {
    WebDriverManager.chromedriver().setup();
    DriverManager.setDriver(new ChromeDriver());
}

@AfterMethod
public void tearDown() { DriverManager.quit(); }
Never use a static WebDriver field in parallel tests. Multiple threads sharing one driver instance causes race conditions and random test failures.
49 What is the difference between XPath axes — child, descendant, following-sibling, parent?
AxisDescriptionExample
child::Direct children only//div/child::input

50 Quick fire — 10 commonly asked short questions
  1. What is a WebElement? — An interface representing an HTML element found on the page.
  2. Can Selenium handle CAPTCHA? — No. Ask devs to disable it in test environments or use a CAPTCHA bypass service.
  3. What is the default polling interval in Explicit Wait? — 500 milliseconds.
  4. How do you clear a text field? — element.clear() or Ctrl+A then Delete via sendKeys(Keys.chord(Keys.CONTROL, "a"), Keys.DELETE).
  5. What is driver.manage().timeouts().pageLoadTimeout()? — Sets max time to wait for a page to load completely. Default is 300 seconds.
  6. How do you verify the page title? — Assert.assertEquals(driver.getTitle(), "Expected Title")
  7. What does driver.getWindowHandle() return? — A unique string ID for the current browser window.
  8. How do you maximize the browser? — driver.manage().window().maximize()
  9. What is the difference between isDisplayed() and isEnabled()? — isDisplayed() checks visibility; isEnabled() checks if it can be interacted with (not disabled).
  10. How do you get the value of an input field? — element.getAttribute("value") (not getText() — that's for visible text).
