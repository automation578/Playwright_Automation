1. API & REST Fundamentals

Conceptual questions always asked first
1 What is an API? What is a REST API?
API (Application Programming Interface) — A contract that defines how two software systems communicate. It exposes certain functions/data so external systems can use them without knowing the internal implementation.

REST (Representational State Transfer) — An architectural style for designing networked APIs using HTTP. A REST API:
  • Uses standard HTTP methods (GET, POST, PUT, DELETE)
  • Is stateless — each request contains all needed information; server stores no session
  • Uses URLs (endpoints) to represent resources
  • Exchanges data in JSON or XML format
  • Returns standard HTTP status codes
Example: GET https://api.example.com/users/1 — fetches user with ID 1.
2 What are the 6 REST constraints (principles)?
  1. Client-Server — Client and server are separate. Client handles UI; server handles data. They communicate only through the API.
  2. Stateless — Server stores no client session. Every request must be self-contained with all needed info (auth token, parameters).
  3. Cacheable — Responses should indicate whether they can be cached. Caching reduces load and improves performance.
  4. Uniform Interface — Consistent URL structure, standard HTTP methods, self-descriptive messages.
  5. Layered System — Client doesn't know if it's talking directly to the server or through proxies/load balancers.
  6. Code on Demand (optional) — Server can send executable code (JavaScript) to the client.

5 What is the difference between URI, URL, and endpoint?
  • URI (Uniform Resource Identifier) — General identifier for a resource. Can be a URL or URN.
  • URL (Uniform Resource Locator) — A URI that specifies how to access the resource (protocol + location). e.g., https://api.example.com/users/1
  • Endpoint — In API context, the specific URL + HTTP method combination that exposes a resource. e.g., GET /users/1 is an endpoint.

Idempotent = calling the same request multiple times produces the same result. GET, PUT, DELETE are idempotent. POST is NOT — multiple POSTs create multiple resources.
7 What is the difference between PUT and PATCH?
  • PUT — Replaces the entire resource. If you omit a field, it gets deleted or reset to default. Idempotent.
  • PATCH — Updates only the specified fields. Other fields remain unchanged.
// Existing user: { "name": "John", "email": "john@test.com", "role": "user" }

// PUT — replaces entire resource
PUT /users/1
{ "name": "John Updated" }
// Result: { "name": "John Updated" }  ← email and role are GONE

// PATCH — updates only specified field
PATCH /users/1
{ "name": "John Updated" }
// Result: { "name": "John Updated", "email": "john@test.com", "role": "user" }
8 What are the HTTP status code ranges? List important ones.

Key distinctions:
  • 200 vs 201 — 200: request succeeded. 201: resource was created (POST response).
  • 204 — Success but no response body (DELETE success).
  • 401 vs 403 — 401: not authenticated (no/invalid token). 403: authenticated but not authorized (wrong permissions).
  • 404 vs 410 — 404: resource not found (may exist later). 410 Gone: permanently deleted.
  • 500 vs 503 — 500: server bug. 503: server down/overloaded (temporary).

10 What is the difference between Query Parameters and Path Parameters?
  • Path Parameters — Part of the URL path, used to identify a specific resource. Required. e.g., /users/{id} → /users/123
  • Query Parameters — Appended after ?, used for filtering, sorting, pagination. Optional. e.g., /users?page=2&sort=name&role=admin

// Query param — HOW you want it (filter/sort/page)

GET /products?category=electronics&minPrice=100&sort=price_asc

3. Java 11 HttpClient — Setup & Basics

Built-in Java HTTP client — no third-party library needed
11 What is the Java HttpClient? Why was it introduced?
java.net.http.HttpClient was introduced in Java 11 (as a stable API after a preview in Java 9). Before it, Java only had HttpURLConnection which was verbose and cumbersome.

Features:
  • Supports HTTP/1.1 and HTTP/2
  • Both synchronous (send()) and asynchronous (sendAsync()) requests
  • Built-in support for WebSocket
  • Fluent builder API — much cleaner code
  • No external dependency needed (unlike Apache HttpClient or OkHttp)
12 What are the three core components of Java HttpClient?
  • HttpClient — The main client instance. Configure timeouts, redirects, auth, proxy. Reuse across requests.
  • HttpRequest — Represents a single HTTP request: method, URL, headers, body.
  • HttpResponse<T> — Represents the server response: status code, headers, body.
import java.net.http.HttpClient;
import java.net.http.HttpRequest;
import java.net.http.HttpResponse;

// 1. Create client (reuse this across tests)
HttpClient client = HttpClient.newBuilder()
    .connectTimeout(Duration.ofSeconds(10))
    .followRedirects(HttpClient.Redirect.NORMAL)
    .build();

// 2. Build request
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://reqres.in/api/users/1"))
    .GET()
    .header("Accept", "application/json")
    .timeout(Duration.ofSeconds(15))
    .build();

// 3. Send and get response
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());

// 4. Assert
System.out.println(response.statusCode());  // 200
System.out.println(response.body());        // JSON string
13 What are BodyHandlers and BodyPublishers?
  • BodyHandlers — Tell HttpClient how to handle the response body. Used in client.send(request, handler).
  • BodyPublishers — Define the request body content. Used when sending POST/PUT requests.
// BodyHandlers — response body handling
HttpResponse.BodyHandlers.ofString()        // body as String
HttpResponse.BodyHandlers.ofInputStream()    // body as InputStream
HttpResponse.BodyHandlers.ofByteArray()      // body as byte[]
HttpResponse.BodyHandlers.ofFile(path)       // save body to file
HttpResponse.BodyHandlers.discarding()       // ignore body (DELETE)

// BodyPublishers — request body content
HttpRequest.BodyPublishers.ofString("{ \"name\": \"John\" }")  // String body
HttpRequest.BodyPublishers.ofFile(path)                        // file body
HttpRequest.BodyPublishers.ofByteArray(bytes)                  // byte[] body
HttpRequest.BodyPublishers.noBody()                            // no body (GET)
14 What is the difference between synchronous send() and asynchronous sendAsync()?
  • send() — Blocks the calling thread until the response arrives. Simple, straightforward.
  • sendAsync() — Returns a CompletableFuture<HttpResponse<T>> immediately. The calling thread continues; a background thread handles the response. Useful for non-blocking / parallel API calls.
// Synchronous — blocks thread
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());
System.out.println(response.statusCode()); // prints after response arrives

// Asynchronous — non-blocking
client.sendAsync(request, HttpResponse.BodyHandlers.ofString())
    .thenApply(HttpResponse::body)
    .thenAccept(System.out::println)
    .join(); // wait for completion in tests

// Parallel API calls — send multiple requests simultaneously
CompletableFuture<HttpResponse<String>> req1 = client.sendAsync(request1, HttpResponse.BodyHandlers.ofString());
CompletableFuture<HttpResponse<String>> req2 = client.sendAsync(request2, HttpResponse.BodyHandlers.ofString());
CompletableFuture.allOf(req1, req2).join(); // wait for both

15 How do you write a complete GET request with Java HttpClient?
import java.net.URI;
import java.net.http.*;
import java.time.Duration;

public class GetUserTest {
    private static final HttpClient client = HttpClient.newBuilder()
        .connectTimeout(Duration.ofSeconds(10))
        .build();

    public static void main(String[] args) throws Exception {

        // Build request
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://reqres.in/api/users/2"))
            .GET()
            .header("Accept", "application/json")
            .build();

        // Send request
        HttpResponse<String> response = client.send(
            request, HttpResponse.BodyHandlers.ofString()

        // Validate
        System.out.println("Status: " + response.statusCode()); // 200
        System.out.println("Body: "   + response.body());

        // Assertions (with TestNG/JUnit)
        assert response.statusCode() == 200 : "Expected 200";
        assert response.body().contains("Janet") : "Name not found";
    }
}

17 How do you write PUT, PATCH, and DELETE requests?
// PUT — full update
HttpRequest putRequest = HttpRequest.newBuilder()
    .uri(URI.create("https://reqres.in/api/users/2"))
    .PUT(HttpRequest.BodyPublishers.ofString("{ \"name\": \"Jane\", \"job\": \"Lead\" }"))
    .header("Content-Type", "application/json")
    .build();

// PATCH — partial update (HttpClient has no .PATCH() shortcut)
HttpRequest patchRequest = HttpRequest.newBuilder()
    .uri(URI.create("https://reqres.in/api/users/2"))
    .method("PATCH", HttpRequest.BodyPublishers.ofString("{ \"job\": \"Senior Lead\" }"))
    .header("Content-Type", "application/json")
    .build();

HttpResponse<String> patchResp = client.send(patchRequest, HttpResponse.BodyHandlers.ofString());
// 200 OK

// DELETE — no body usually
HttpRequest deleteRequest = HttpRequest.newBuilder()
    .uri(URI.create("https://reqres.in/api/users/2"))
    .DELETE()
    .build();

HttpResponse<Void> deleteResp = client.send(deleteRequest, HttpResponse.BodyHandlers.discarding());
System.out.println(deleteResp.statusCode()); // 204 No Content
18 How do you read response headers in Java HttpClient?
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());

// Get a specific header value
String contentType = response.headers().firstValue("content-type").orElse("not found");
System.out.println(contentType); // application/json; charset=utf-8

// Get all values of a header (multi-value headers)
List<String> cookies = response.headers().allValues("set-cookie");

// Get all headers as a Map
Map<String, List<String>> allHeaders = response.headers().map();
allHeaders.forEach((name, values) ->
    System.out.println(name + ": " + String.join(", ", values))
);

5. JSON Parsing — Jackson & Gson

Converting between JSON strings and Java objects
19 What is Jackson? How do you parse JSON with it?
Jackson is the most widely used Java JSON library. It provides ObjectMapper for serialization (Java → JSON) and deserialization (JSON → Java). 
// Maven dependency
<dependency>
  <groupId>com.fasterxml.jackson.core</groupId>
  <artifactId>jackson-databind</artifactId>
  <version>2.17.0</version>
</dependency>
import com.fasterxml.jackson.databind.ObjectMapper;
import com.fasterxml.jackson.databind.JsonNode;

ObjectMapper mapper = new ObjectMapper();

// 1. Parse JSON string to JsonNode (tree model — no POJO needed)
String json = response.body();
JsonNode root = mapper.readTree(json);

// Navigate the tree
String name   = root.path("data").path("first_name").asText();
int    id     = root.path("data").path("id").asInt();
String email  = root.at("/data/email").asText(); // JSON Pointer syntax

// 2. Deserialize to POJO
User user = mapper.readValue(json, User.class);

// 3. Serialize POJO to JSON string
User newUser = new User("John", "QA Lead");
String jsonBody = mapper.writeValueAsString(newUser);

// 4. Deserialize to List
List<User> users = mapper.readValue(json, mapper.getTypeFactory()
    .constructCollectionType(List.class, User.class));
20 How do you create a POJO for JSON deserialization?
// User.java — matches this JSON:
// { "id": 1, "first_name": "George", "email": "george@reqres.in" }

import com.fasterxml.jackson.annotation.JsonIgnoreProperties;
import com.fasterxml.jackson.annotation.JsonProperty;

@JsonIgnoreProperties(ignoreUnknown = true)  // ignore extra JSON fields
public class User {
    private int id;

    @JsonProperty("first_name")  // map snake_case JSON to camelCase Java
    private String firstName;

    private String email;

    // Default constructor (required by Jackson)
    public User() {}

    // Getters and setters
    public int getId()           { return id; }
    public String getFirstName() { return firstName; }
    public String getEmail()     { return email; }
    public void setId(int id)                { this.id = id; }
    public void setFirstName(String n)       { this.firstName = n; }
    public void setEmail(String e)           { this.email = e; }
}

24 How do you get a token from a login API and use it in subsequent requests?
public class AuthUtils {
    private static final HttpClient client = HttpClient.newHttpClient();
    private static final ObjectMapper mapper = new ObjectMapper();

    // Step 1: Call login endpoint to get token
    public static String getToken() throws Exception {
        String loginBody = "{ \"email\": \"test@test.com\", \"password\": \"Test@123\" }";

        HttpRequest loginReq = HttpRequest.newBuilder()
            .uri(URI.create("https://reqres.in/api/login"))
            .POST(HttpRequest.BodyPublishers.ofString(loginBody))
            .header("Content-Type", "application/json")
            .build();

        HttpResponse<String> loginResp = client.send(loginReq, HttpResponse.BodyHandlers.ofString());

        // Parse token from response: { "token": "QpwL5tpe..." }
        return mapper.readTree(loginResp.body()).path("token").asText();
    }
}

26 Show a complete CRUD test using RestAssured.
import io.restassured.RestAssured;
import static io.restassured.RestAssured.*;
import static org.hamcrest.Matchers.*;

public class UserApiTest {

    @BeforeClass
    public static void setup() {
        RestAssured.baseURI = "https://reqres.in";
        RestAssured.basePath = "/api";
    }

    // GET — with assertions
    @Test
    public void getUser() {
        given()
            .header("Accept", "application/json")
        .when()
            .get("/users/2")
        .then()
            .statusCode(200)
            .body("data.id",         equalTo(2))
            .body("data.first_name", equalTo("Janet"))
            .body("data.email",      containsString("@reqres.in"))
            .time(lessThan(3000L));
    }

    // POST — create and extract ID
    @Test
    public void createUser() {
        String body = "{ \"name\": \"QA Bot\", \"job\": \"Tester\" }";

        String userId = given()
            .contentType("application/json")
            .body(body)
        .when()
            .post("/users")
        .then()
            .statusCode(201)
            .body("name", equalTo("QA Bot"))
            .body("id",   notNullValue())
        .extract().path("id");

        System.out.println("Created user ID: " + userId);
    }

    // DELETE — verify 204
    @Test
    public void deleteUser() {
        given()
        .when()
            .delete("/users/2")
        .then()
            .statusCode(204);
    }
}
27 How do you use JSONPath in RestAssured?
// JSONPath in .body() assertions
.then()
    .body("data.id", equalTo(2))                    // exact value
    .body("data.email", containsString("@"))         // contains
    .body("data", notNullValue())                      // not null
    .body("data.first_name", equalToIgnoringCase("janet"))

// Extract a value

    .then().extract().path("data.email");

// Get full response as JsonPath
Response response = given().when().get("/users?page=1");
List<String> emails = response.jsonPath().getList("data.email");
int totalPages = response.jsonPath().getInt("total_pages");

// Array assertions
.body("data", hasSize(6))
.body("data.id", hasItems(1, 2, 3))
.body("data.email", everyItem(containsString("@reqres.in")))
28 How do you validate a JSON schema in RestAssured?
// src/test/resources/schemas/user-schema.json
{
  "type": "object",
  "properties": {
    "data": {
      "type": "object",
      "properties": {
        "id":    { "type": "integer" },
        "email": { "type": "string", "format": "email" },
        "first_name": { "type": "string" }
      },

}
import static io.restassured.module.jsv.JsonSchemaValidator.matchesJsonSchemaInClasspath;

@Test
public void validateUserSchema() {
    given().when().get("/users/2")
        .then()
        .statusCode(200)
        .body(matchesJsonSchemaInClasspath("schemas/user-schema.json"));
}
Schema validation is powerful — it verifies the structure and data types of the response, not just values. Catches API contract breaks early.

// BaseApi.java — centralized client config
public abstract class BaseApi {
    protected static final HttpClient client = HttpClient.newBuilder()
        .connectTimeout(Duration.ofSeconds(15))
        .build();
    protected static final String BASE_URL = ConfigReader.get("apiBaseUrl");
    protected static String authToken;

    @BeforeClass
    public static void init() throws Exception {
        authToken = AuthUtils.getToken();
    }

    protected HttpRequest.Builder baseRequest(String path) {
        return HttpRequest.newBuilder()
            .uri(URI.create(BASE_URL + path))
            .header("Authorization", "Bearer " + authToken)
            .header("Content-Type",  "application/json")
            .header("Accept",        "application/json");
    }
}
30 What should you validate in an API test?
For a thorough API test, validate:
  1. Status code — Is it the expected code? (200, 201, 204, 400, 404…)
  2. Response body — field values — Does the data match what was sent/expected?
  3. Response body — data types — Is ID an integer, not a string? Schema validation.
  4. Response headers — Content-Type: application/json? Authentication headers present?
  5. Response time — Is the API responding within acceptable SLA? (e.g., < 2000ms)
  6. Error messages — For 4xx responses, is the error message descriptive and correct?
  7. Negative scenarios — Missing required fields → 400. Wrong auth → 401. Invalid ID → 404.
  8. Boundary conditions — Empty string, max length, negative numbers, null values.

32 How do you test a REST API that requires OAuth 2.0 authentication?
OAuth 2.0 uses a token-based flow. The most common grant type for testing is Client Credentials (machine-to-machine) or Resource Owner Password (for test users).

Steps:
  1. Call the token endpoint with client credentials to get an access token
  2. Store the token (it expires — handle refresh)
  3. Pass it as Authorization: Bearer {token} in every API call
// Step 1: Get OAuth token (client credentials flow)
String tokenBody = "grant_type=client_credentials"
    + "&client_id="     + ConfigReader.get("clientId")
    + "&client_secret=" + ConfigReader.get("clientSecret")
    + "&scope=api.read";

HttpRequest tokenReq = HttpRequest.newBuilder()
    .uri(URI.create("https://auth.example.com/token"))
    .POST(HttpRequest.BodyPublishers.ofString(tokenBody))
    .header("Content-Type", "application/x-www-form-urlencoded")
    .build();

HttpResponse<String> tokenResp = client.send(tokenReq, HttpResponse.BodyHandlers.ofString());
String accessToken = mapper.readTree(tokenResp.body()).path("access_token").asText();
33 How do you test an API for pagination?
// Test first page
given()
    .queryParam("page", 1)
    .queryParam("per_page", 6)
.when()
    .get("/users")
.then()
    .statusCode(200)
    .body("page",         equalTo(1))
    .body("per_page",     equalTo(6))
    .body("data",         hasSize(6))
    .body("total",        greaterThan(0))
    .body("total_pages",  greaterThan(0));

// Test last page — should not have more items than per_page
Response firstPage = given().queryParam("page",1).get("/users");
int totalPages = firstPage.jsonPath().getInt("total_pages");

given()
    .queryParam("page", totalPages)
.when()
    .get("/users")
.then()
    .statusCode(200)
    .body("data.size()", greaterThan(0)); // last page has data

// Test beyond last page — should return 404 or empty
given()
    .queryParam("page", totalPages + 1)
.when()
    .get("/users")
.then()
    .body("data", hasSize(0)); // or .statusCode(404)
34 How do you test negative API scenarios?
// 1. Missing required field → 400 Bad Request
given()
    .body("{ \"name\": \"\" }") // missing 'job'
.when()
    .post("/users")
.then()
    .statusCode(400)
    .body("error", containsString("job is required"));

// 2. Invalid ID → 404 Not Found
given().when()
    .get("/users/9999")
.then()
    .statusCode(404);

// 3. Missing auth token → 401 Unauthorized
given()
    .header("Authorization", "") // no token
.when()
    .get("/secure-endpoint")
.then()
    .statusCode(401)
    .body("error", equalTo("Missing authentication token"));

// 4. Invalid data type → 422 Unprocessable
given()
    .body("{ \"age\": \"not-a-number\" }")
.when()
    .post("/users")
.then()
    .statusCode(422);

// 5. Expired token → 401
given()
    .header("Authorization", "Bearer EXPIRED_TOKEN")
.when()
    .get("/users")
.then()
    .statusCode(401);
35 Quick-fire: 10 short API testing questions
  1. What is the difference between 401 and 403? — 401: Not authenticated (no/bad token). 403: Authenticated but not authorized (wrong role/permission).
  2. What is idempotency? — Calling the same request multiple times produces the same result. GET, PUT, DELETE are idempotent. POST is not.
  3. What is Content-Type vs Accept header? — Content-Type: format of the request body you're sending. Accept: format you expect in the response.
  4. What is CORS? — Cross-Origin Resource Sharing. Browser security policy that blocks requests from a different origin. APIs allow it via Access-Control-Allow-Origin response header.
  5. What is the difference between JSON and XML? — JSON: lightweight, readable, array support, language-agnostic. XML: verbose, supports attributes, schemas (XSD), namespaces. REST uses JSON; SOAP uses XML.
  6. What is HTTP/2? — The second major version of HTTP. Supports multiplexing (multiple requests over one connection), header compression, server push. Java HttpClient supports it.
  7. What is a mock API? — A simulated API that returns predefined responses. Used in testing when the real API isn't available or to test specific edge cases. Tools: WireMock, MockServer.
  8. What is Postman used for in API testing? — Manual API testing, collections, environments, automated test scripts (Newman), documentation generation, and team collaboration.
  9. What does the Accept: application/json header do? — Tells the server the client expects a JSON response. If server can't provide JSON, it may return 406 Not Acceptable.
  10. How do you measure API response time in RestAssured? — .then().time(lessThan(2000L)) — asserts response is under 2 seconds. Or use response.getTime() to get the value.
